case-study — offline-edge-ai

Offline Edge AI Medical Intelligence

Atian.ai · prototype demonstrated live in a real ambulance
Whisper.cppMedGemma 4BARM64 / Dockeraiohttp + SSEfully offline

Ambulance crews document patient encounters by hand, after the fact — and field connectivity is too unreliable (and patient audio too sensitive) to depend on the cloud. This prototype runs the entire pipeline on edge hardware inside the vehicle: an Ericsson Cradlepoint R1900 cellular router transcribes crew–patient audio with Whisper.cpp while an edge PC runs MedGemma 4B to generate clinical documentation in real time. No internet, no cloud — patient audio never leaves the vehicle.

what it does
On the router (ARM64, 4 cores, 4 GB RAM): a MediaMTX RTSP broker ingesting the vehicle mic (12 MB RAM) and a Whisper.cpp STT container with medical prompt biasing, dictionary correction and noise filtering — 39 MB total edge footprint
On the edge PC: an aiohttp dashboard server (encounter state machine, keyword alerting, diarization, SSE push, SQLite) plus MedGemma 4B via Ollama running a 17-task EMS pipeline — SOAP notes, Patient Care Report, and MIST-format ED handoff generated before the ambulance arrives
Frontend: a real-time clinical dashboard with live transcription, alerts, the classification pipeline, and a clinical AI chat answering questions against the live encounter
engineering highlights
Fit Whisper into 27 MB of RAM by replacing PyTorch (2.5 GB image) with Whisper.cpp (500 MB image) after profiling showed the router couldn't hold it
Whisper prompt-bias + a domain dictionary beat an on-router LLM for medical term correction — and the LLM was starving the router's CPU, causing the connectivity drops we'd been chasing
10-hour continuous stress test: 1,015 transcriptions at 103/hour sustained, no restarts
Everything air-gapped: models, inference, storage and UI served from the two edge devices over the local LAN

case-study — epcr-autofill-agent

Autonomous ePCR Auto-Fill Agent

safety-gated browser agent · human-in-the-loop

A safety-gated browser agent that turns AI-extracted patient-care data into a pre-filled, QA-validated ePCR chart — cutting EMT documentation time while keeping every clinical decision human.

Playwright-over-CDP agent filling a major ePCR vendor's charts: a 287-field specification across 12 chart pages, auto-generated by a recon crawler that expands every popup and picker
11 widget-level fill adapters for the vendor's legacy UI — masked time inputs, coded-picker modals, iframe scopes — each with tri-state result logging for auditability
Agency QA rules encoded as a deterministic validator producing per-field gap reports (rule code + exact fix) — the safety net that lets automation write without ever locking or submitting a chart
Clinical safety by design: never synthesizes values, never creates patient records, dry-run mode, idempotent audit trail, deterministic runtime — LLMs only at build time
Containerized "Chromium-as-a-service" (Chrome + Xvfb + noVNC + CDP): the EMT watches the very browser the agent drives, with an injected in-page control overlay
